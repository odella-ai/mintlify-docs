---
title: "Memory & Reminders"
description: "Control long-term memory, overnight review, and follow-up tracking for a worker."
---

**Memory & Reminders** controls how a worker remembers useful context and tracks future commitments.

## Enable dreaming

When enabled, the worker reviews recent activity overnight and saves useful context to long-term memory.

Dreaming helps preserve details such as:

- User preferences
- Decisions made in conversations
- Useful context from recent work
- Patterns the worker should remember later

## Dream diary

Controls the kind of diary entry created after overnight review.

| Mode | What it does |
| --- | --- |
| **Simple notes** | Adds a short, structured entry. |
| **Detailed notes** | Creates richer notes using a more detailed review process. |

Dreaming is designed to organize information over time, not replace governed knowledge bases.

## Remember follow-ups

When enabled, the worker tracks commitments such as:

- “Check back tomorrow”
- “Reply if they do not respond”
- “Send the summary next week”

This helps the worker resurface pending work at the right time instead of losing it in conversation history.

<Tip>
Enable follow-up memory for workers that manage conversations, inboxes, customer requests, or recurring stakeholder updates.
</Tip>
